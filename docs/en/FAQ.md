# Frequently Asked Questions (FAQ)

## General

### What is the purpose of this website?
The website serves as a professional portfolio to showcase skills, experiences, and projects in infrastructure engineering, virtualization, software development, and data analysis.

### How can I view the website?
You can access the website online at **[umeglio.github.io](https://umeglio.github.io)** or clone the repository to view it locally.

## Installation and Configuration

### What are the prerequisites?
- Git (to clone the repository)
- A text editor (optional, for modifications)
- A modern browser

### How do I set up the website locally?
Follow the [Installation Guide](INSTALLATION.md) for detailed step-by-step instructions.

### Can I use the website offline?
Yes, you can download the repository and view the HTML files locally in your browser.

## Modification and Customization

### How do I modify my profile?
Edit the `index.html` (Italian) and `index_en.html` (English) files. See the [Usage Guide](USAGE.md) for details on each section.

### How do I add a new skill?
Modify the "skills" section in the HTML files. Add a new element with the skill name and percentage.

### Can I modify the colors and style?
Yes, CSS is included in the HTML files. You can modify properties in the `<style>` tag to customize the appearance.

### How do I change the profile image?
Replace the `avatar.png` file with your image, keeping the same filename and ensuring it's in the main directory.

## Languages

### Does the website support languages other than English and Italian?
Currently, it only supports Italian (index.html) and English (index_en.html). To add other languages, see the [Contributor Guide](CONTRIBUTING.md).

### How do I keep different language versions in sync?
When making changes, update both `index.html` and `index_en.html` with corresponding content. Keep the same HTML structure to ease maintenance.

## Publishing and Hosting

### Where is the website hosted?
The website is hosted on **GitHub Pages** at the domain `umeglio.github.io`.

### How long does it take for the website to update after I push?
Usually within 1-2 minutes. If you don't see changes, try clearing your browser cache (Ctrl+Shift+Delete).

### Can I use a custom domain?
Yes, you can set up a custom domain through the repository GitHub settings. For details, see [GitHub Pages documentation](https://docs.github.com/en/pages).

### Is the website secure?
Yes, GitHub Pages provides HTTPS by default. Since the website contains static content, it is inherently secure.

## Contributing

### How can I contribute to the project?
See the [Contributor Guide](CONTRIBUTING.md) for complete details.

### Can I add a library or framework?
The project intentionally doesn't use external frameworks to maintain simplicity and performance. Open an issue to discuss if you think it's necessary.

### How do I report a bug?
Open a [GitHub Issue](https://github.com/umeglio/umeglio.github.io/issues) with a detailed description of the problem and steps to reproduce it.

## Technical

### What technologies does the website use?
- HTML5 for structure
- CSS3 for styling and responsive layout
- Vanilla JavaScript for animations and interactivity
- GitHub Pages for hosting

### Why doesn't it use a modern framework?
To ensure fast loading, simple maintenance, and no external dependencies. A static portfolio website doesn't need one.

### Is the website responsive?
Yes, the website is fully responsive and optimized for desktop, tablet, and mobile devices.

### How do animations work?
Animations are implemented using CSS3 and vanilla JavaScript. You'll see them during page scrolling and interactions.

## Accessibility

### Is the website accessible?
Yes, the website is designed to comply with WCAG accessibility guidelines.

### How do I report accessibility issues?
Open a GitHub Issue describing the accessibility problem you encountered.

## Performance

### How can I check the website's performance?
You can use tools like:
- Google PageSpeed Insights
- GTmetrix
- WebPageTest

Visit one of these sites and enter the website URL.

### Is the website slow?
If the website loads slowly:
1. Clear your browser cache
2. Try a different browser
3. Check your internet connection speed
4. Open an issue if the problem persists
